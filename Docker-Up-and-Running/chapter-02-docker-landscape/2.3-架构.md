# 2.3 架构

> **后端学习提示**：本章**必学**。建立 Registry、C/S 架构、分层镜像、无状态设计等全局观；是 Ch04 镜像与 Ch08 Compose 的理论基础。

* **核心主旨与实用要点**
  * 剖析 Docker 在底层强悍技术之上的极简架构模型，重点拆解其客户端-服务端（C/S）工作方式、网络通信接口及 API 设计。
* **详细知识点拆解**
  * **客户端-服务器模式**：
    * Docker 基础架构由三部分组成：**客户端（Client）**、**服务器（Daemon 守护进程）**以及可选的**注册处（Registry）**。
    * *易错细节*：Docker 不区分客户端和服务器的二进制文件，它们使用的是**同一个**可执行文件（`docker`）。当执行带有 `-d` 参数的命令时，即启动为监听后台的守护进程模式。
  * **网络端口和 UNIX 套接字**：
    * *默认通信机制*：守护进程默认监听一个本地 **UNIX 套接字**进行进程间通信。
    * *TCP 端口规范*：在向 IANA 注册规范后，Docker 默认监听未加密的 **2375** 端口和加密的 **2376** 端口，替代了早期非官方的 4243 端口。
  * **应用程序编程接口（API）**：
    * Docker 守护进程对外暴露了功能丰富的 **RESTful Web API**。
    * *实际应用*：借助 API（如 Go 或 Ruby 语言的封装库），可以轻易将 Docker 的功能集成到自动化脚本、部署工具或调度平台中，使得工具的二次开发极为便捷。
  * **容器的网络层**：
    * *机制原理*：Docker 会创建一个虚拟以太网桥（通常命名为 `docker0`），并将宿主机的端口绑定到容器上。
    * *性能表现*：流经 `docker0` 虚拟网桥的入站流量仅经过一次网络地址转换（NAT），因此虽然是容器，但网络性能极高，吞吐量极大。

| 组件 | 作用 | 后端常用命令 |
|------|------|--------------|
| Client | 发指令 | `docker build/run/push` |
| Daemon | 管理容器/镜像/网络 | 后台服务 |
| Registry | 存储分发镜像 | `docker pull/push` |
