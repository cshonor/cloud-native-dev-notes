# 1.5 总结 (Summary)

* **核心结论**：Kubernetes从根本上旨在改变云端应用程序的构建和部署方式。
* **关键主旨**：其核心设计目标是为开发者带来前所未有的开发速度（Velocity）、效率（Efficiency）和敏捷性（Agility）。

> 💡 **后续拓展空间**：直接承接后续关于容器构建的基础（如Docker镜像分层原理）及Pod调度机制。

## 本章速记

| 概念 | 一句话 |
|------|--------|
| Velocity | 高可用前提下持续交付新功能的能力 |
| 不可变基础设施 | 更新 = 替换新镜像，旧镜像可回滚 |
| 声明式配置 | 描述期望状态，配合 Git 做 IaC |
| 自愈 | 持续对齐当前状态与期望状态 |
| 解耦 | API + 负载均衡器隔离组件与团队 |
| 抽象化 | 容器 API 屏蔽机器/云差异，提升可移植性 |
| 效率 | 同置装箱 + Namespace 降本，支撑 CI 全量部署 |
| KaaS | 小团队优先托管 K8s；大团队可自建 |
